# Hub Console

The **Hugr Hub management console** — a standalone SPA (platform admin + agent
fleet + skills + monitoring) plus an **embeddable agent-chat microfrontend**.
Design 009; the generation brief is
[`../design/009-management-console/claude-design-prompt.md`](../design/009-management-console/claude-design-prompt.md).

## Layout

```
console/
├── embed.go        # Go: embeds dist/ and serves it (go:embed all:dist)
├── dist/           # Vite build output (committed placeholder index.html only)
│   ├── index.html  # placeholder until the SPA is built; build overwrites it
│   └── .gitignore  # ignores real build output, keeps the placeholder
└── <SPA source>    # generated by claude design (package.json, src/, vite.config.ts, index.html …)
```

The Go binary serves this at **`/console/`**; a public runtime-config endpoint
is at **`/console/config.json`**. Both are exempt from the auth middleware (the
static assets and pre-login config are public); the SPA authenticates its own
`/hugr` and `/api/v1/*` calls with the user's OIDC token.

## Build contract (for the generated SPA)

- **Vite `base: '/console/'`** and **`build.outDir: 'dist'`** — hub embeds
  `console/dist`. Deep links resolve via SPA-fallback in `embed.go`.
- **Runtime config, not build-time:** fetch `GET /console/config.json` →
  `{ oidc_issuer, oidc_client_id, oidc_scopes, api_base }`. Do not hardcode the
  issuer/client — they vary per deployment.
- **Auth:** OIDC Authorization Code + PKCE against `oidc_issuer`; send the
  access token as `Authorization: Bearer` on every `/hugr`, `/api/v1/*`,
  `/skills/*` request. **Chat SSE must be read via `fetch()` + `ReadableStream`**
  (native `EventSource` cannot set the `Authorization` header).
- **Chat is a separable package** exposing `mountChat(el, props)` /
  `<hub-chat>` so the same bundle also mounts in a JupyterLab Lumino panel.

## Server-side config (hub env)

The OIDC issuer + public client id are **discovered from Hugr** — hub fetches
Hugr's public `GET /auth/config` (which returns the browser-reachable `issuer` +
`client_id`) and serves them at `/console/config.json`. Nothing is pinned per
deployment, and Hugr works with any OIDC provider. The override knobs below stay
empty by default.

| Env | Default | Meaning |
|---|---|---|
| `HUB_CONSOLE_ENABLED` | `true` | Serve `/console`. |
| `HUB_CONSOLE_DIR` | `` (embedded build) | Serve the SPA from this on-disk directory instead of the embedded build. The container builds the SPA and drops it here, so the Go binary and the console ship/update independently. A dir with no `index.html` falls back to the embedded build. |
| `HUB_CONSOLE_OIDC_ISSUER` | `` (discover from Hugr `/auth/config`) | Override the discovered browser-reachable OIDC issuer. |
| `HUB_CONSOLE_OIDC_CLIENT_ID` | `` (discover from Hugr `/auth/config`) | Override the discovered public PKCE client id. |
| `HUB_CONSOLE_OIDC_SCOPES` | `openid profile email` | Requested scopes. |
| `HUB_CONSOLE_API_BASE` | `` (same origin) | API base if the SPA is served off-origin. |

Hub reaches Hugr's `/auth/config` at `HUGR_URL` (its `/ipc` suffix trimmed).

## Building

```sh
pnpm install          # once
pnpm dev              # dev server on :5199 (proxies /hugr /api /skills → localhost:15000)
pnpm build            # → dist/ (embedded by the Go binary via go:embed)
pnpm build:chat       # → dist-chat/ (the separable chat microfrontend bundle)
```

Open `http://localhost:5199/console/?demo=1` to run fully offline (seeded mock
data, OIDC bypassed). Drop `?demo=1` to hit a real hub.

**Docker:** `Dockerfile.hub-service` builds the SPA in a `node:22` stage and
copies `console/dist` into the final image at `/usr/local/share/hub-console`,
setting `HUB_CONSOLE_DIR` so the binary serves it from disk. The Go build only
`go:embed`s the committed `dist/index.html` placeholder (the disk build is the
real one; the placeholder is just the fallback), so the Go build no longer
depends on the SPA build and the two stages run in parallel.

## Chat microfrontend

`src/chat` is a self-contained, framework-agnostic package: it never imports the
SPA router or globals — everything arrives through props. Public API (`src/chat/index.ts`):

```ts
import { mountChat } from './chat'
const handle = mountChat(element, {
  apiBase: '',                          // same origin
  getToken: async () => currentBearer,  // OIDC access token
  chatId, agentId, theme, demo, showRail,
})
handle.unmount()
```

There is also a `<hub-chat api-base chat-id agent-id theme demo token>` custom
element. A JupyterLab Lumino `Widget` embeds it by calling `mountChat(this.node,
props)` in `onAfterAttach` and `handle.unmount()` in `dispose`, loading the
`dist-chat` bundle + its CSS. The SPA consumes `ChatApp` directly (see
`src/screens/ChatScreen.tsx`).
